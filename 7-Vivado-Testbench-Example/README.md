# Dual-Port FIFO Simulation with Tcl & SystemVerilog

This repository demonstrates a **complete FPGA simulation workflow** for a simple Dual-Port FIFO.  
It teaches **Tcl scripting for simulation automation** and uses **SystemVerilog testbench** to interface with the DUT.  

---

## 📂 Repository Structure

```text
 ├─ src/                  # Your RTL source files
 │   └─ dual_port_fifo.v
 ├─ tb/                   # Testbench files
 │   └─ tb_dual_port_fifo.sv
 ├─ run_sim_fifo.tcl      # Main Tcl test runner
 ├─ logs/                 # Logs generated per testcase
 └─ fifo.vcd              # Waveform generated by simulation
```
## How to Run
vivado -mode batch -source ./run_sim_fifo.tcl

# Test Flow

## Compile RTL and Testbench
The Tcl script compiles both `dual_port_fifo.v` and `tb_dual_port_fifo.sv`.

## Elaborate Design
Vivado elaborates the DUT and testbench.

## Multiple Testcases
The Tcl script runs multiple test scenarios such as:

- Sequential write/read (0–15)
- Random write/read sequences
- Full FIFO check

## Stimulus from Tcl
All write/read operations are driven by Tcl using `force` commands, keeping the testbench passive.

## Logging & Debugging

- Each testcase generates a log file in `logs/`
- Console outputs provide real-time status and mismatch warnings
- Waveform saved as `fifo.vcd` for GTKWave or Vivado GUI inspection

## Reset and FIFO Status
The testbench initializes reset and monitors FIFO `empty` and `full` flags automatically.

---
## Why Use Tcl Automation?

- Reproducible results: Every run executes the same test flow
- Regression testing: Easy to add more testcases without modifying the testbench
- Version control friendly: Only RTL, SV testbench, and Tcl scripts are tracked
- Flexible stimulus: Modify sequences directly from Tcl without touching SV
